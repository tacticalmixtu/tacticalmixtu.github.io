Dynamic Programming

August 31, 2020

What is dynamic programming#

  • Two properties
    • optimal sub-structure
      • an optimal solution to the problem could be built from optimal solutions to sub-problems
    • overlapping sub-problems, could be avoided by memorization

How to solve dynamic programming problem#

  • Define problem: from states conclude problem definition
    • for each states that would vary among sub-problems, define a corresponding parameter for recursion function
  • Define recurrence relationship: from choices conclude recurrence relationship
    • from optimal sub-structure, constructs recurrence relationship
    • consider
      • what choice could be made
      • how do states change due to this choice
  • Define memorization
    • each parameter in recursive function becomes a dimension in dp table, dp(i) becomes dp[i]
  • Bottom-up implementation
    • consider dependency relationship in dp table: in order to compute dp[i], it relies on dp[i-1]
    • Optional: there might be cases when we could reuse one dimension to decrease space complexity

Knapsack#

  • Normal Knapsack(with repetition)

    • For a knapsack with capacity of 10, items(weight, value) are: (6, $30), (3, $14), (4, $16), (2, $9)

  • Define problem:

    • $K(w)$: maximum value achievable with a knapsack of capacity w

  • Sub-problem:

    • $K(w) = max{K(w-w_i)+v_i}$, for each $w_i<w$

  • Find out repeated computation, use memorization to avoid, complete recursion tree is not fully revealed

    • K(10)
      • K(10-6) + 30
        • K(4-3) + 14 + 30
        • K(4-2) + 9 + 30
          • K(2-2) + 9 + 9 + 30
      • K(10-3) + 14
        • K(7-6) + 30 + 14
        • K(7-4) + 16 + 14
        • K(7-3) + 14 + 14
        • K(7-2) + 9 + 14
          • K(5-4) + …
      • K(10-4)…
      • K(10-2)…

  • Implementation:

    • compute from K(1) up to K(w) for a given w; Since K(w) only relies on $K(w_i)$ that have been computed before $K(w)$
func main() {
    values := []int{10, 40, 50, 70}
    weights := []int{1, 3, 4, 5}
    fmt.Println(knapsack(8, values, weights))
    // Output: for capacity 8, use one item of weight 3 and the other of weight 5
    // 110
}

func knapsack(w int, values, weights []int) int {
    table := make([]int, w+1)
    choice := make([][]int, w+1)
    for i := 1; i < len(table); i++ {
        // localMax := table[i-1]
        for j := range values {
            weight, value := weights[j], values[j]
            if weight > i {
                continue
            }
            if table[i] < table[i-weight]+value {
                table[i] = table[i-weight] + value
                choice[i] = append(choice[i], weight)
            }
        }
    }
    fmt.Println(choice)
    return table[w]
}

Coin change#

  • 322. Coin change
    • Define problem: it requires at least dp(i) coins to to constitute total amount of i
    • Define recurrence relationship: $dp(i) = min{dp(i-c)+1,c<=i}$, c is the coin value
func coinChange(coins []int, amount int) int {
  if len(coins) < 1 || amount < 0 {
    return -1
  }
  
  table := make([]int, amount+1)
  // base case
  table[0] = 0
  // initialize as maximum value, for this problem, should be amount+1
  for i := 1; i < len(table); i++ {
    table[i] = amount + 1
  }

  for i := 1; i < len(table); i++ {
    for _, coin := range coins {
      if i < coin {
        continue
      }
      if table[i] > 1+table[i-coin] {
        table[i] = 1 + table[i-coin]
      }
    }
  }
  // check if there's a valid solution
  if table[amount] == amount+1 {
    return -1
  }
  return table[amount]
}
  • 518. Coin change 2
    • from states conclude problem definition:
      • states
        • which coins are selected
        • current amount that could be constituted
      • problem definition: there are $dp(i,j)$ combinations that use only first $i$ coins could constitute amount of $j$
    • from choices conclude recurrence relationship:
      • choice
        • select coin of value $c$: $dp(i,j) = dp(i,j-c)$, only when $c<=j$
        • does not select current coin: $dp(i,j)=dp(i-1,j)$
      • recurrence relationship: we want all possible combinations, therefore $dp(i,j)=dp(i,j-c)+dp(i-1,j)$, for each $c<=j$
func change(amount int, coins []int) int {
  table := make([][]int, len(coins)+1)
  for i:=range table{
      table[i] = make([]int,amount+1)
      // initialize all dp[i][0] since for amount=0, only 1 solution is there
      table[i][0] = 1
      // for dp[0][j] all should be 0, since no coins are there to provide a combination
  }
  for i:=1;i<=len(coins);i++{
      for j :=1;j<=amount;j++{
          if coins[i-1] <= j{
              table[i][j] = table[i][j-coins[i-1]] + table[i-1][j]
          }else{
              table[i][j] = table[i-1][j]
          }
      }
  }
  return table[len(coins)][amount]
}
  • State space reduction:
    • since $dp(i,)$ only relies on $dp(i-1,)$
    • therefore dimension of $i$ could be reused
func change(amount int, coins []int) int {
  table := make([]int, amount+1)
  table[0] = 1
  for i := range coins {
    for j := 1; j <= amount; j++ {
      // table[i][j] would automatically be table[i-1][j] if c<=j, no update would be performed
      if coins[i] <= j {
        table[j] += table[j-coins[i]]
      }
    }
  }
  return table[amount]
}
